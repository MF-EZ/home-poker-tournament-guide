# Contributing to Home Poker Tournament Guide

Thank you for your interest in contributing to this project!

This repository exists to provide a **simple and practical guide** for organizing home poker tournaments. The goal is to help people run smooth, fun tournaments without overcomplicating things.

---

## Repository Purpose

This guide is designed for:
- **Casual and home games** (not casino-level operations)
- **Simplicity over complexity**
- **Practical advice** that works in real-world settings
- Keeping tournaments **easy and fun to manage**

All contributions should align with these principles.

---

## How You Can Contribute

Contributions are welcome in the following areas:

### 1. **Improving Existing Content**
- Clarify explanations or fix errors
- Add examples that make concepts easier to understand
- Improve formatting or structure for better readability

### 2. **Adding Practical Tips**
- Share proven strategies from real home tournaments
- Suggest blind structures that work well
- Recommend simple tools or approaches that reduce complexity

### 3. **Fixing Issues**
- Correct typos or grammar mistakes
- Fix broken links
- Update outdated information

### 4. **Documentation Enhancements**
- Add helpful sections that stay true to the guide's simplicity
- Include FAQs based on common questions
- Suggest improvements to organization or flow

---

## What We're **Not** Looking For

To keep this guide focused, we generally avoid:
- Overly complex tournament structures
- Advanced poker strategy or theory
- Casino or professional tournament advice
- Tools or systems that require significant setup
- Anything that makes running a home tournament harder

---

## Contribution Guidelines

### Before You Start
1. Check if a similar issue or suggestion already exists
2. Make sure your contribution aligns with the repository's purpose
3. Keep changes simple and well-explained

### Making Changes
1. **Fork the repository** and create a new branch
2. Make your changes with clear, concise commits
3. Test any links or examples you add
4. Keep the tone casual and accessible

### Submitting Your Contribution
1. Open a **pull request** with a clear description
2. Explain **why** the change improves the guide
3. Reference any related issues

### Review Process
- Contributions will be reviewed for clarity and alignment with the guide's goals
- Feedback may be provided to help refine the contribution
- Simple, well-explained improvements are typically merged quickly

---

## Style and Tone

When contributing, please match the existing style:
- **Casual and friendly** — like explaining to a friend
- **Practical and direct** — focus on what works
- **Simple language** — avoid jargon when possible
- **Organized and scannable** — use headers and lists

---

## Questions or Suggestions?

If you're unsure whether a contribution fits, feel free to:
- Open an issue to discuss your idea
- Ask questions before making changes

We appreciate your interest in making home poker tournaments easier and more fun for everyone!
